# Wealth Engine

**Wealth Engine** is a Streamlit-based stock strategy workstation for optimizing, saving, monitoring, and exporting stock trading plans.

It combines a strategy engine (`wealth_engine.py`) with a clean Streamlit interface (`app.py`) so you can:

- Run strategy optimization on a selected stock.
- Save approved results as official plans.
- Separate saved plans into **My Stock** and **Watchlist**.
- Refresh current signals safely, including a Free API Safe Refresh mode.
- Review dashboards, current trades, latest signals, replay results, and exported reports.
- Download clean Excel, JSON, CSV, and ZIP packages for each saved symbol.

> **Important:** This project is an analysis and tracking tool. It is not financial advice.

## 3. Key Concepts

### Optimize

Searches for the best strategy settings over the selected date range and timeframe.

### Signal

Uses a saved plan and refreshes the current decision using the latest available market data.

### Replay

Re-runs a saved plan over historical data to rebuild the trade history and recent events.

### My Stock vs Watchlist

Both sections use full saved plans. The difference is only classification and display.

| Section | Meaning |
|---|---|
| `My Stock` | Core saved plans you actively follow. |
| `Watchlist` | Saved plans you want to monitor separately. |

A symbol can be moved between sections without changing or deleting its plan, reports, or packages.

## 6. Quick Start

1. Open the app.
2. Go to **Analyze Stock**.
3. Enter a symbol, date range, timeframe, investment amount, trials, jobs, and minimum trades.
4. Run the analysis.
5. Save the result to either:

```text
Save to My Stock
Save to Watchlist
```

After saving, the symbol becomes an official saved plan and appears in dashboards, signal pages, reports, and auto refresh.

```bash
python wealth_engine.py IREN --mode optimize --start-date 2025-01-01 --end-date 2026-04-28 --investment-amount 500 --timeframes 4h --trials 1000 --jobs 8 --min-trades 12 --adjust splits --output results
```


## 11. Trades Pages

### Trades Of My Stock

Tracks current trade status for symbols in My Stock.

### Trades Of Watchlist

Tracks current trade status for symbols in Watchlist.

The displayed risk and reward values are calculated relative to the entry price:

```text
Risk to Stop % = (Stop / Entry - 1) * 100
Reward to Target % = (Target / Entry - 1) * 100
```

---

## 14. Auto Refresh

Auto Refresh updates saved plans only. It does not optimize new settings and does not add new symbols.

### Fast Refresh

Use only if your API plan can handle multiple symbols quickly.

### Free API Safe Refresh

Recommended for free Twelve Data API usage. It refreshes symbols one by one and waits between symbols to reduce credit-limit failures.

Default safe settings:

```text
Credits / minute: 8
Pause inside symbol: 5 to 8 seconds
Wait after symbol: 60 to 75 seconds
Buffer seconds: 10
```

If API credit errors return, increase:

```text
Pause inside symbol
Wait after symbol
```

## 15. Background Jobs

Analyze Stock runs in the background so you can move between pages while optimization continues.

Background job files are stored in:

```text
state/current_job.json
state/current_job.log
state/background_job_runner.py
```

If a job gets stuck, stop Streamlit and remove:

```text
state/current_job.json
state/current_job.log
state/last_analysis.json
state/background_job_runner.py
```

Then restart the app.

## 16. API Key Safety

Before uploading the project to GitHub or deploying it online, do **not** expose private API keys inside the code.

Recommended production approach:

```text
Use environment variables or Streamlit secrets.
```

Example local environment variable:

```bash
set TWELVEDATA_API_KEY=your_key_here
```

Recommended code pattern:

```python
import os
API_KEY = os.getenv("TWELVEDATA_API_KEY", "")
```

### Recommended Free Deployment: Streamlit Community Cloud

Best first choice for this project because it runs Streamlit apps directly from a GitHub repository.

Suggested repository structure:

```text

тФЬтФА results/
тФВ  тФФтФА .gitkeep
тФЬтФА packages/
тФВ  тФФтФА .gitkeep
тФЬтФА exports/
тФВ  тФФтФА .gitkeep
тФЬтФА state/
тФВ  тФФтФА .gitkeep
тФФтФА archive/
   тФФтФА .gitkeep
```

Deployment steps:

1. Create a GitHub repository.
2. Upload the project files.
3. Add `.gitkeep` files inside empty folders.
4. Add sensitive values as secrets, not inside code.
5. Deploy `app.py` from Streamlit Community Cloud.
6. Test one stock with low trials.
7. Test Save to My Stock, Refresh Signal, Run Replay, and Download Package.
8. Enable Free API Safe Refresh only after confirming the app runs normally.

### Important Hosting Note

Free hosting is good for demos and personal usage, but it may sleep, restart, or clear generated files depending on the platform. For a production version that must refresh 24/7 and preserve reports reliably, use persistent storage or a VPS.
